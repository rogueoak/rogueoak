# 0002 - Build plan: pull-request CI

Source spec: `docs/specs/0002-pr-ci.md`. Reference: matthewmaynes.com
`.github/workflows/{verify,ci}.yml` and `.github/dependabot.yml`.

## Step 1 - Reusable verify workflow

File: `.github/workflows/verify.yml`.
- `name: verify`, `on: workflow_call: {}`, `permissions: contents: read`.
- Job `verify` on `ubuntu-latest`: checkout -> setup-node (node 24, `cache: npm`) -> `npm ci` ->
  `npm run lint` -> `npm run build` -> `npm test`.
- Pin actions to SHAs with version comments.

## Step 2 - PR gate

File: `.github/workflows/ci.yml`.
- `name: ci`, `on: pull_request: branches: [main]`, `permissions: contents: read`.
- `concurrency: { group: ci-${{ github.event.pull_request.number }}, cancel-in-progress: true }`.
- Job `verify` that `uses: ./.github/workflows/verify.yml`.

## Step 3 - Dependabot

File: `.github/dependabot.yml` - `github-actions`, directory `/`, weekly, commit prefix `ci`.

## Step 4 - Verify locally

- Confirm the npm scripts referenced (`lint`, `build`, `test`) exist in `package.json`.
- Run `npm ci && npm run lint && npm run build && npm test` to prove the gate passes on this branch.
- Lint the workflow YAML (actionlint if available, else a YAML parse check).

## Step 5 - Commit, PR, review

- Conventional Commit (`ci: ...` / `feat: ...`), open PR.
- Persona review: architect (CI structure, reusable workflow boundary), security (SHA-pinned
  actions, workflow permissions, no secrets). engineer/tester have little surface (no app code),
  scope them out unless the diff warrants.
- Address findings, merge on green review.
